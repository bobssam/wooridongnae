공통 프로토콜

[apiName]/**

프로토콜

공통 HTTP 프로토콜 규약입니다.
일부 에러나 문제에 대해선 프로토콜로 선처리 합니다.

400 - 잘못된 요청

경로는 존재하나, 메소드나 파라메터등이 잘못 기입되거나 누락되었을때 발생합니다.

401 - 권한 없음

기능에 대한 권한이 없을때 발생합니다.

403 - 접근 권한 없음

세션이 해제되거나, 인과된 아이피나 아이디가 아닐때 발생합니다.
재로그인을 유도합니다.

404 - 페이지 없음

요청 경로가 잘못되었을때 입니다.

405 - 메소드 허용 안됨

사용할수 없는 메소드를 요청할때 발생합니다.

500 - 서버 내부 오류

프로그램 상의 오류가 있을때 발생합니다.
